# Superpowers Workflow

You must follow a strict, disciplined development methodology for every task:

1. **Brainstorming & Requirements**: Clarify the user's request and gather all necessary context. Do not write code yet.
2. **Planning**: Create a detailed architectural plan and implementation strategy.
3. **Test-Driven Development (TDD)**: Write failing tests for the planned changes *before* writing the implementation code.
4. **Implementation**: Write the actual code to make the tests pass.
5. **Systematic Review**: Review your own code, run the tests, and verify the fix before finalizing the task.

**CRITICAL RULE**: Do not bypass these steps. If you attempt to write implementation code before tests, you are violating this framework. Acting like a disciplined senior engineer is required.
